#Q2: local storage ,section storage ,cookies differences

localStorage, sessionStorage, and cookies are web browser storage mechanisms that store data locally on the user's device. They differ in scope, storage limits, and data management. localStorage persists data even after the browser is closed and can be accessed across browser sessions. sessionStorage is available only for the duration of the page session. cookies are used for storing small data such as user preferences or session identifiers and can have an expiration time set by the server. localStorage provides larger storage capacity compared to cookies and sessionStorage, but browsers may impose their own limits. cookies have smaller storage limits and can be accessed and manipulated both on the client and server side.

#Q3:How Event loop work?

Summary: In JavaScript, the event loop manages asynchronous operations by continuously checking the event queue for events. When the main thread is idle, it processes these events, executing their associated callback functions. This allows JavaScript to handle asynchronous tasks efficiently while maintaining responsiveness in web applications, as it ensures that asynchronous operations do not block the main thread.

#Q5:How Choose Quiz function work?

            
                const chooseQuiz = (index, givenAns) => {
                    const isExist = answers.find((ans) => ans.id === quizData[index].id);
                    if (isExist) {
                      let serial = 0;

                          break;
                        }
                        serial++;
                      }

                      answers.push({ ...quizData[index], givenAns });
                    }

                  };
The chooseQuiz function manages user quiz selections by determining whether an answer is already available for a specific quiz index. It updates the answer with the user's new choice if one already exists. The user's choice is added to the responses array to create a new answer in the event that none already exists. The function uses the displayAnswers function to cause the updated answers to be displayed on the user interface (UI) after updating or adding the answer.
